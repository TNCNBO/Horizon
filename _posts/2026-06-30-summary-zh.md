## [最高法院限制地理围栏令。](https://www.theguardian.com/us-news/2026/jun/29/supreme-court-geofence-warrants-case-decision) ⭐️ 8.0/10

美国最高法院裁定，索取位置数据的地理围栏令会触发宪法隐私保护，并驳回了政府关于搜索范围太小或地点太公开而不涉及第四修正案的主张。该案涉及 Okello Chatrie，警方曾利用 Google 在弗吉尼亚州里士满一起银行抢劫案附近的位置历史数据锁定他。 这一裁决可能显著限制警方在尚不知道嫌疑人身份时，如何从 Google 等公司获取批量位置记录。它也在手机、应用和商业数据库让位置追踪变得日常化的背景下，加强了数字隐私法理。 根据报道的事实，Google 最初提供了 19 个匿名账户的列表，这些账户关联的设备在抢劫案前后一个小时内位于银行 150 米范围内，随后调查人员进一步缩小了名单。Elena Kagan 大法官撰写的多数意见据称强调，即使只获取相对很小一部分手机位置数据，也仍可能构成第四修正案意义上的搜索。

hackernews · Hacker News 热门 · 6月29日 15:54 · [社区讨论](https://news.ycombinator.com/item?id=48720924)

**背景**: 地理围栏令也称为反向位置令，是要求公司找出在特定时间段内位于特定地理区域的设备或账户。与针对已知嫌疑人的传统令状不同，这种方法先从地点和时间出发，再搜索数据库来寻找可能相关的人。争议在于，这类令状可能同时收集许多并未被怀疑犯罪的旁观者的数据。

## [Fil-C 让 C 上下文切换具备内存安全性。](https://fil-c.org/context_switches) ⭐️ 7.0/10

Fil-C 发布了一篇技术文章，解释它如何在保持低层代码兼容性的同时，让 C 中的 setjmp/longjmp 和 ucontext 等上下文切换机制具备内存安全性。文章提到，ucontext API 支持从 0.680 版本开始新增，目前需要从源码构建 Fil-C 才能使用。 C 中的上下文切换会直接处理已保存的寄存器、栈帧和控制流，因此它是检验内存安全 C 实现的一类高难度场景。如果 Fil-C 能安全支持这些机制，更多现有 C 和 C++ 软件就有可能在不重写的情况下获得内存安全检查。 核心安全问题在于，longjmp 不能恢复一个其原始栈帧已经返回或被覆盖的上下文。Fil-C 通过跟踪已保存的跳转缓冲区或上下文是否仍然有效，在不安全的控制转移会破坏其内存安全模型时触发 panic。

hackernews · Hacker News 热门 · 6月30日 00:38 · [社区讨论](https://news.ycombinator.com/item?id=48727177)

**背景**: Fil-C 是一种具备内存安全性的 C 和 C++ 实现，目标是在零修改或少量修改的情况下运行现有代码，并把内存安全错误捕获为 panic。在 C 中，setjmp 会把当前执行状态的一部分保存到缓冲区，而 longjmp 稍后恢复该状态以执行非局部跳转。ucontext 系列函数包括 getcontext、setcontext、makecontext 和 swapcontext，提供更底层的执行上下文保存与切换能力，常被用来构建类似协程或纤程的系统。

## [美团发布 LongCat-2.0。](https://longcat.chat/blog/longcat-2.0/) ⭐️ 7.0/10
