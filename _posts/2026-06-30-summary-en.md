## [Supreme Court limits geofence warrants.](https://www.theguardian.com/us-news/2026/jun/29/supreme-court-geofence-warrants-case-decision) ⭐️ 8.0/10

The U.S. Supreme Court ruled that geofence warrants seeking location data trigger constitutional privacy protections, rejecting the government’s argument that the search was too limited or too public to implicate the Fourth Amendment. The case involved Okello Chatrie, who was identified after police used Google location-history data near a Richmond, Virginia bank robbery. The decision could significantly restrict how police obtain bulk location records from companies such as Google when they do not yet know a suspect’s identity. It also strengthens digital privacy doctrine at a time when phones, apps, and commercial databases make location tracking routine. According to the reported facts, Google initially provided a list of 19 anonymized accounts associated with devices within 150 meters of the bank during a one-hour window around the robbery, after which investigators narrowed the list further. Justice Elena Kagan’s majority opinion reportedly emphasized that even a relatively small slice of cellphone location data can still be a Fourth Amendment search.

hackernews · Hacker News 热门 · Jun 29, 15:54 · [Discussion](https://news.ycombinator.com/item?id=48720924)

**Background**: A geofence warrant, also called a reverse location warrant, asks a company to identify devices or accounts that were within a defined geographic area during a defined time period. Unlike a traditional warrant aimed at a named suspect, this method starts with a place and time and then searches a database to find possible people. The controversy is that such warrants can sweep in data about many bystanders who are not suspected of any crime.

## [Fil-C makes C context switching memory-safe.](https://fil-c.org/context_switches) ⭐️ 7.0/10

Fil-C published a technical article explaining how it makes C context-switching mechanisms such as setjmp/longjmp and ucontext memory-safe while preserving compatibility with low-level code. The article notes that ucontext API support is new since release 0.680 and currently requires building Fil-C from source. Context switching in C directly manipulates saved registers, stack frames, and control flow, which makes it a difficult test case for any memory-safe C implementation. If Fil-C can support these mechanisms safely, more existing C and C++ software can potentially run with memory-safety checks without being rewritten. The core safety issue is that longjmp must not restore a context whose original stack frame has already returned or been overwritten. Fil-C addresses this by tracking whether a saved jump buffer or context is still valid and panicking when unsafe control transfer would violate its memory-safety model.

hackernews · Hacker News 热门 · Jun 30, 00:38 · [Discussion](https://news.ycombinator.com/item?id=48727177)

**Background**: Fil-C is a memory-safe implementation of C and C++ that aims to run existing code with zero or minimal changes while catching memory-safety errors as panics. In C, setjmp saves part of the current execution state into a buffer, and longjmp later restores that state to perform a non-local jump. The ucontext family, including getcontext, setcontext, makecontext, and swapcontext, provides lower-level APIs for saving and switching execution contexts, often used to build coroutine or fiber-like systems.
